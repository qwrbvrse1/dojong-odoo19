## Stage 5 — Onboarding lifecycle steps + trial conversion (change 8)

Goal: `bash scripts/usability_pass/verify/s5.sh` passes. Read the gate FIRST. It mandates
exact names — these are the contract:

- New boolean fields on `dojo.onboarding.record`:
  `step_trial_booked`, `step_waiver_signed`, `step_intro_completed`,
  `step_membership_activated`, `step_uniform_issued`.
  Keep the legacy data-entry step fields for compatibility; `progress_pct` and
  `missing_steps` now compute over the NEW lifecycle set.
- Method `dojo.onboarding.record._sync_derived_steps()`:
  `step_waiver_signed` ⇐ member has a signed waiver (`waiver_signed_on` /
  `has_signed_waiver` from dojo_sign); `step_membership_activated` ⇐
  `membership_state == 'active'`; `step_trial_booked` ⇐ membership_state in
  ('trial','active') or a CRM trial booking exists. `step_intro_completed` and
  `step_uniform_issued` are manual-only. State flips to `completed` only when ALL five are
  true; otherwise stays/returns to `in_progress`. New records default to `in_progress`.
- The onboarding wizard (`_create_student_member`) STOPS writing `state='completed'` — it
  creates the record `in_progress` and calls `_sync_derived_steps()`.
- Trial conversion on `dojo.member`: new fields `converted_from_trial` (Boolean) and
  `trial_converted_on` (Datetime). `action_set_active()` sets both when the previous state
  was `'trial'`.
- Kiosk `perform_onboarding_action` (`dojo_kiosk/models/dojo_kiosk_service.py:1790`)
  `complete_step` accepts the new step keys (manual ones at minimum:
  `intro_completed`, `uniform_issued`) and the kiosk workflow_status step list reflects the
  lifecycle set.
- Migration for existing records: a post-init/migration hook maps legacy records — derived
  steps synced from current member state; `intro_completed`/`uniform_issued` set True for
  records previously `completed` (grandfathering); recompute state.

The gate proves behavior via odoo shell: creates a member + record, asserts default
`in_progress`; signs the waiver fields directly and asserts `_sync_derived_steps()` flips
`step_waiver_signed`; sets membership trial → `action_set_active()` and asserts both
conversion fields; asserts the kiosk service accepts `complete_step` with
`step_key='intro_completed'`.

Upgrade `dojo_onboarding` (+ `dojo_kiosk`, `dojo_core` if touched), restart web before gating.

Done = gate passes + commit.
