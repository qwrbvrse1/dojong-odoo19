## Stage 4 — Demo-flow Playwright suite (DEMO_RESCUE.md Phase 4)

Goal: `bash scripts/demo_rescue/verify/s4.sh` passes. That gate (a) executes
`verify/smoke/run.sh` — which YOU must author — and (b) independently hits the kiosk page and
surname search API. Read the gate first.

Author the suite at `verify/smoke/` with entrypoint `verify/smoke/run.sh` (exit 0 = pass).
Plain Playwright node scripts (like `scripts/demo_rescue/verify/admin_smoke.mjs` — reuse its
error-collection pattern) are preferred over a test-runner setup. The suite must cover, against
http://localhost:8070:

1. All five demo logins reach a working page (no error dialog, no /web/login bounce-back).
2. Admin: open Members list and one member form; open Kiosk config. No JS errors.
3. Instructor (backend): sees their assigned sessions; does NOT see Credits or admin Onboarding
   menus; opening their dashboard produces no AccessError.
4. Kiosk (token URL from `dojo_kiosk_config`): page renders; search "Smi" shows Smith results;
   search "Doe" shows Doe.
5. Kiosk instructor mode (PIN 123456): active session auto-selected ("Active:" pill visible);
   roster tiles render with photos and at least one workflow badge; open a member profile —
   Onboarding/Waiver/Membership/Grading cards render; Manage tab: mark one onboarding step done
   → success state and progress changes; mark one member present → tile state persists after a
   page reload.
6. Zero uncaught JS errors on every screen above (reuse the IGNORE filter from admin_smoke.mjs).

If a suite step fails because the APP is broken (not the test), fix the app — smallest change
that makes the real flow work; revert to `main`'s version of a file when faster. Selector
brittleness is yours to solve (prefer role/text selectors; add `data-*` hooks to kiosk templates
only if unavoidable).

Done = gate passes + commit.
