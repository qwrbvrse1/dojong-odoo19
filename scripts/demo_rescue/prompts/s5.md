## Stage 5 — Freeze & runbook (DEMO_RESCUE.md Phase 5)

Goal: `bash scripts/demo_rescue/verify/s5.sh` passes. That gate cold-restarts the stack
(`docker compose down && up`) and re-runs EVERY prior gate against the fresh containers, then
checks `DEMO_RUNBOOK.md`.

Your tasks:
1. Write `DEMO_RUNBOOK.md` in repo root: start command, all five demo logins + passwords, kiosk
   URL + PIN 123456, the seed re-center command (from stage 3) to run right before the demo so
   session times are live, and a 5-line demo flow (admin view → kiosk surname search →
   instructor mode: auto-selected class, roster badges, mark attendance, complete onboarding
   step).
2. Run the gate yourself. The cold restart is the point — if anything only worked because of
   warm state (un-upgraded module, manual tweak in a running container, data not committed by a
   seed script), it will fail here. Fix forward by making the seed/upgrade scripts own that
   state.
3. `git add -A && git commit`, then `git tag -f demo-ready`.

Done = gate passes + commit + tag.
